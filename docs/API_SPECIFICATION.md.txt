# Authentication Module

| Method | Endpoint | Authentication | Purpose |
|---------|----------|---------------|----------|
| POST | /auth/register | ❌ Public | Register a new user |
| POST | /auth/login | ❌ Public | Login and issue Access Token |
| POST | /auth/refresh | Refresh Cookie | Generate new Access Token |
| POST | /auth/logout | ✅ Required | Logout and invalidate Refresh Token |
| GET | /users/me | ✅ Required | Get current user profile |
| PATCH | /users/me | ✅ Required | Update user profile |
| PATCH | /users/me/password | ✅ Required | Change password |

---

## Register Request

```json
{
  "fullName": "Rupesh Vishwakarma",
  "email": "rupesh@gmail.com",
  "phone": "9876543210",
  "password": "Password@123"
}
```

### Response

```json
{
  "message": "Registration successful"
}
```

### HTTP Status

```
201 Created
```

---

## Login Response

```json
{
  "accessToken": "jwt-token",
  "tokenType": "Bearer",
  "expiresIn": 900,
  "role": "USER"
}
```

---

## Refresh Token Policy

- Stored as HttpOnly Cookie
- Secure Cookie
- SameSite=Strict
- Expiry = 7 Days

---

## Access Token Policy

- JWT Bearer Token
- Expiry = 15 Minutes

# Complaint Module

| Method | Endpoint | Authentication | Purpose |
|---------|----------|---------------|----------|
| POST | /complaints | ✅ Required | Create Complaint |
| GET | /complaints | ❌ Public | Public Complaint Listing |
| GET | /complaints/{id} | Optional* | Complaint Details |
| GET | /users/me/complaints | ✅ Required | Current User Complaints |
| PATCH | /complaints/{id} | ✅ Required | Update Complaint |
| DELETE | /complaints/{id} | ✅ Required | Soft Delete Complaint |
| POST | /complaints/{id}/vote | ✅ Required | Add Vote |
| DELETE | /complaints/{id}/vote | ✅ Required | Remove Vote |

---

## Complaint Details Access Rules

Authentication is optional.

- If Complaint Status = **APPROVED**
  - Accessible publicly.

- Otherwise
  - Accessible only to:
    - Complaint Owner
    - Administrator


---

## Update Complaint

```
PATCH /complaints/{id}
```

Allowed Fields

- title
- description
- categoryId

Restrictions

- Images cannot be modified in Version 1.
- Only Complaint Owner may update.
- Complaint Status must be SUBMITTED.

---

## Delete Complaint

```
DELETE /complaints/{id}
```

Behavior

- Soft Delete
- Owner Only
- Status must be SUBMITTED

---

## Voting

```
POST /complaints/{id}/vote
```

Rules

- One user can vote only once.
- Vote can be removed.

# Validation Rules

## User Registration

- Full Name → Required, Maximum 100 Characters
- Email → Required, Valid, Unique
- Phone → Required, Unique, Regex: `^[6-9]\d{9}$`
- Password → Minimum 8 Characters with:
  - Uppercase Letter
  - Lowercase Letter
  - Number
  - Special Character

---

## Complaint

- Title → Required, Maximum 150 Characters
- Description → Required
- Category → Required
- Latitude → Required
- Longitude → Required
- At least one image is required

---

# Business Rules

## Authentication

- JWT Access Token
- Refresh Token stored in HttpOnly Cookie
- Role-Based Authorization

## Visibility Rules

### Public Users

Can view only:

- APPROVED Complaints
- Not Soft Deleted

---

### Citizens

Can view their own:

- SUBMITTED
- UNDER_REVIEW
- APPROVED
- REJECTED
- RESOLVED
- Soft Deleted

---

### Administrators

Can access all complaints.

---

## Edit Rules

Citizen can edit only if:

- Status = SUBMITTED

---

## Delete Rules

Citizen can delete only if:

- Status = SUBMITTED

Deletion type:

- Soft Delete

---

## Status History

```

```
