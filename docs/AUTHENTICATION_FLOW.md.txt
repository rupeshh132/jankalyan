# Registration Flow

Endpoint

POST

/api/v1/auth/register

Validation

- Full Name Required
- Email Required
- Unique Email
- Phone Required
- Unique Phone
- Password Policy

Process

1. Validate request
2. Check duplicate email
3. Check duplicate phone
4. Encrypt password using BCrypt
5. Save user
6. Return success response

Response

201 Created

---

# Login Flow

Endpoint

POST

/api/v1/auth/login

Validation

- Email Required
- Password Required

Process

1. Validate credentials
2. Load user
3. Verify password
4. Generate Access Token
5. Generate Refresh Token
6. Save Refresh Token
7. Return JWT
8. Set Refresh Token as HttpOnly Cookie

Response

200 OK

---

# JWT Strategy

Algorithm

HS256

Token Type

Bearer

Contains

- User ID
- Email
- Role

Expiration

15 Minutes

Header

Authorization

Bearer <token>

---

# Refresh Token Strategy

Storage

Database

+

HttpOnly Cookie

Cookie Settings

HttpOnly = true

Secure = true

SameSite = Strict

Expiration

7 Days

---

# Refresh Flow

Endpoint

POST

/api/v1/auth/refresh

Process

1. Read Refresh Cookie
2. Validate Token
3. Check Expiry
4. Verify Database Record
5. Generate New Access Token
6. Return New JWT

No new Refresh Token issued unless rotation is enabled in future versions.

---

# Logout Flow

Endpoint

POST

/api/v1/auth/logout

Process

1. Authenticate User
2. Delete Refresh Token
3. Clear Cookie
4. Return Success

---

# Password Encryption

# Security Rules

Never

- Store JWT in Local Storage
- Store password in plain text
- Return refresh token in JSON response
- Hardcode JWT secret

Always

- Use BCrypt
- Use HttpOnly Cookie
- Validate every request
- Verify user role
