# Architecture Rules

- Feature-first package structure
- No cross-feature business logic
- DTO Pattern
- Repository Pattern
- Service Layer
- Global Exception Handling
- Response Wrapper
- Bean Validation

---

# Prohibited

Do NOT

- Create random packages
- Put business logic inside controllers
- Access repositories directly from controllers
- Duplicate DTOs
- Duplicate utilities
- Rename existing packages

---

# Compliance

This folder structure is fully compatible with

- Spring Boot 3.x
- Java 21
- React
- Modular Monolith Architecture
