# VALIDATION_RULES.md

# Jankalyan Validation Rules

Version: 1.0

Status: 🔒 FROZEN

---

# Purpose

This document defines all validation rules used throughout the application.

Frontend and Backend validations must remain consistent.

Server-side validation is mandatory.

Client-side validation is only for better user experience.

---

# General Rules

Always

- Trim leading/trailing spaces
- Reject null values for required fields
- Reject empty strings
- Validate before saving to database

Never

- Trust client-side validation
- Accept malformed data

---

# User Registration Validation

## Full Name

Required

Minimum Length

2 Characters

Maximum Length

100 Characters

Allowed

- Alphabets
- Spaces

Not Allowed

- Numbers
- Special Characters

Example

✅ Rupesh Vishwakarma

❌ Rupesh123

---

## Email

Required

Maximum Length

255 Characters

Rules

- Valid email format
- Unique
- Converted to lowercase before saving

Example

✅ user@example.com

❌ user@

---

## Mobile Number

Required

Regex

```
^[6-9]\d{9}$
```

Length

10 Digits

Unique

Yes

Example

✅ 9876543210

❌ 1234567890

---

## Password

Required

Minimum Length

8 Characters

Maximum Length

20 Characters

Must Contain

- One Uppercase Letter
- One Lowercase Letter
- One Number
- One Special Character

Regex

```
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&]).{8,20}$
```

Example

✅ Password@123

❌ password

---

## Confirm Password

Required

Must exactly match Password
